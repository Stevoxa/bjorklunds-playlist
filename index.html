Lokal klient · PKCE · krypterad lagring

Klistra in låtlista

En rad per låt. Tips: Artist – Titel, Artist, Titel eller Titel av Artist. Sök på Spotify tolkar listan och hämtar träffar (kräver inloggning under Inställningar).

Match: Spotify kan returnera flera närliggande spår för samma sökning — välj vilket spår som ska användas (album och år visas så likadana titlar skiljs åt). Välj: använd reglaget om sökningen ska tas med när du skapar eller uppdaterar spellistan (du kan lämna det av trots träff).

Matchningar

Välj
Spotify API-logg (sökning + spelliste-skrivning)

Sökning: JSON-rad utan kind. Skapa/lägga till/ersätta spellista: rad med kind: "playlist_write" — metod, path, request (t.ex. namn/publik, playlistId, uriCount, uriSample max 8 st), httpStatus, samt vid JSON-svar spotifyError, snapshotId, ev. playlistId från svar. Icke-JSON-svar: bodyPreview. Tokens och client secret loggas inte; känsliga nycklar maskeras.

Fältet Lösenfras för lokal kryptering nedan är bara för att låsa upp/spara krypterad data på din enhet — det är inte ditt Spotify-lösenord.

Varje plats du kör sidan från (t.ex. olika GitHub Pages-mappar) behöver sin egen rad i listan.

            Om du får Forbidden (403) när du skapar spellista eller lägger till låtar
            

              

                Ingen lista med kryssrutor för rättigheter? Det är normalt: Spotify kan lägga samtycke i inloggning/avtal/kod utan att visa varje scope på en egen rad. Det viktiga är vad Spotify faktiskt skickar tillbaka i token-svaret. Efter inloggning ska statusraden här visa raden Beviljade rättigheter (från Spotify): … — där måste playlist-modify-public och/eller playlist-modify-private synas för spellistor.
              

              

                Tvinga nytt samtycke mot Spotify: öppna
                spotify.com/account/apps,
                hitta denna app och välj Ta bort åtkomst. Logga sedan in igen via Logga in med Spotify i denna app och Spara lokalt.
              

              

                Vad det betyder: Spotify svarar ”du får inte göra detta” (HTTP 403). Det är inte ett fel i lösenfras-fältet. Vanligast är att din inloggningstoken saknar behörighet för spellistor, eller att du inte fått en ny token efter att appen begär fler rättigheter.

  2. Fyll i Client ID och Lösenfras som tidigare.

  3. Klicka Logga in med Spotify. Du ska då se Spotifys sida (konto + godkänn rättigheter). Om du inte ser någon Spotify-sida kan webbläsaren redan vara inloggad på spotify.com — öppna då accounts.spotify.com/logout i samma webbläsare, gå tillbaka till appen och prova Logga in igen. Appen skickar show_dialog=true så godkännande oftast visas igen.
    4. 
                
  4. Klicka Spara lokalt så den nya token sparas krypterat.
    5. 
                
  5. Gå tillbaka till Låtar och prova Utför igen.

                I Spotify Developer Dashboard (din app): om läget är Development måste ditt Spotify-konto finnas med som testanvändare, annars kan API neka vissa steg. Kontrollera också att Redirect URI fortfarande stämmer exakt med den URI som visas högst upp här.
              

              

                Varför stramar Spotify åt? I Development mode gäller bland annat krav på Spotify Premium för den som använder API:et, begränsning till ett Development-Client-ID per utvecklare, max fem auktoriserade användare per app och ett smalare API-utbud — se Spotifys egen sammanfattning och uppdateringar i
                Update on Developer Access and Platform Security
                (feb 2026). För endpoint- och fältändringar, se
                Web API Changelog – February 2026.
              

              

                Om du ska skapa publik spellista måste du kryssa i ”Publik spellista” och token måste inkludera rättighet för det. Vid tvekan: lämna den urkryssad (privat spellista) och testa igen.
              

            

          

        


        

          
Sökcache

          

            Träfflistor för samma söksträng (inkl. artist/titel-hjälp) sparas i denna webbläsare i 30 minuter så upprepade sökningar mot Spotify minskar.
            Cachen försvinner när du stänger fliken (sessionStorage). Knappen nedan rensar all sparad sökdata manuellt.
